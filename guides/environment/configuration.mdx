# Environment Configuration

Configure your `.env` file to store wallet credentials and network settings.

## Create .env File

Create a file named `.env` in your `~/metahash/` directory:

```bash .env
WALLET_NAME=mywallet
WALLET_HOTKEY=miner1
WALLET_PASSWORD=yourpassword
BITTENSOR_NETWORK=finney
```

<Steps>
  <Step title="Navigate to Project Directory">
    ```bash
    cd ~/metahash
    ```
  </Step>

  <Step title="Create .env File">
    ```bash
    nano .env
    ```
    Or use your preferred text editor.
  </Step>

  <Step title="Add Configuration">
    Paste the configuration above, replacing values with your actual credentials.
  </Step>

  <Step title="Save and Exit">
    In nano: Press `Ctrl+O`, `Enter`, then `Ctrl+X`
  </Step>
</Steps>

## Configuration Parameters

| Parameter | Description | Example |
|-----------|-------------|---------|
| `WALLET_NAME` | Your coldkey wallet name | `mywallet` |
| `WALLET_HOTKEY` | Your hotkey name | `miner1` |
| `WALLET_PASSWORD` | Wallet password for auto-unlock | `yourpassword` |
| `BITTENSOR_NETWORK` | Network to connect to | `finney` |

## Security Best Practices

<Warning>
Never commit your `.env` file to git! Add it to `.gitignore`:
```bash
echo ".env" >> .gitignore
```
</Warning>

<Note>
The `.env` file prevents password prompts when running the miner, allowing for automated restarts.
</Note>

## Common Issues

<AccordionGroup>
  <Accordion title="WALLET_PASSWORD not set">
    **Problem**: Miner prompts for password repeatedly

    **Fix**: Ensure `WALLET_PASSWORD` line is present in `.env`
  </Accordion>

  <Accordion title=".env missing">
    **Problem**: Password prompts appear

    **Fix**: Create `.env` manually in project root (`~/metahash/`)
  </Accordion>

  <Accordion title="Password prompt keeps appearing">
    **Problem**: Running miner from wrong directory

    **Fix**: Always run miner from the directory containing `.env` (i.e., `~/metahash/`)
  </Accordion>
</AccordionGroup>

## Verification

Verify your `.env` file exists and is in the correct location:

```bash
cat ~/metahash/.env
```

You should see your configuration values (passwords will be visible, so be cautious).
