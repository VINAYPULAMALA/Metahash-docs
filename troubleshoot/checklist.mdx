# Final Checklist

Use this checklist to ensure your MetaHash miner is properly configured and running.

## Environment & Software

<Steps>
  <Step title="Python 3.12 Installed">
    ```bash
    python3 --version
    ```
    <Check>Should show Python 3.8 or higher</Check>
  </Step>

  <Step title="Virtual Environment Active">
    ```bash
    which python
    ```
    <Check>Should show path to `.venv/bin/python`</Check>
  </Step>

  <Step title="Dependencies Installed">
    ```bash
    pip list | grep -E "bittensor|async-substrate"
    ```
    <Check>Both packages should be listed</Check>
  </Step>

  <Step title="Bittensor CLI Working">
    ```bash
    btcli --version
    ```
    <Check>Should display version number</Check>
  </Step>
</Steps>

## Network & Connectivity

<Steps>
  <Step title="Port 8091 Open in AWS">
    <Check>AWS Security Group has inbound rule for TCP 8091</Check>

  </Step>

  <Step title="Port Listening Locally">
    ```bash
    ss -tulpn | grep 8091
    ```

  <Step title="UFW Configured (if enabled)">

</Steps>

## Common Post-Setup Issues

<AccordionGroup>
  <Accordion title="Miner runs but no auctions">
    **Wait Time**: Auctions may not start immediately

    **Check**:
    - Port 8091 externally accessible
    - Validators are active on subnet 73
    - Payment validator has stake

    **Timeline**: Can take hours or days for first auction
  </Accordion>

  <Accordion title="High memory usage">
    **Normal**: Python processes can use 500MB-1GB

    **Monitor**: Use `htop` to track over time

    **Fix if growing**: Add swap or restart periodically
  </Accordion>

  <Accordion title="Occasional restarts">
    **Normal**: Some restarts are expected

    **Check**: systemd should auto-restart

    **Investigate if frequent**: Check logs for errors
  </Accordion>
</AccordionGroup>
